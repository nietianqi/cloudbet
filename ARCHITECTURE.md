### 2. 资金管理流程（2.5倍马丁格尔）

```python
# 公式
s0 = r * B * (q - 1) / (q^(N+1) - 1)
stake = s0 * (q ^ loss_streak)

# 参数
r = 0.30    # 单序列占资金30%
q = 2.5     # 倍率
N = 4       # 最大连输次数
```

**示例（100 USDT 余额）：**
```
轮次 | 连败 | 投注金额 | 累计风险 | 占比
-----|------|----------|----------|------
  1  |  0   |  1.38    |  1.38    | 1.4%
  2  |  1   |  3.46    |  4.84    | 4.8%
  3  |  2   |  8.64    |  13.48   | 13.5%
  4  |  3   |  21.60   |  35.08   | 35.1%
  5  |  4   |  50.00   |  85.08   | 85.1% (触顶)
```

### 3. 评分系统（4维度）

```
总分 = 联赛分 + 让球盘分 + 独赢分 + 走势分

1. 联赛分（0-10分）
   - 五大联赛: 10分
   - 欧洲主流: 8分
   - 其他联赛: 5-7分

2. 让球盘分（0-3分）
   - 最优点: 1.70（3分）
   - 偏离惩罚: -10 * |odds - 1.70|

3. 独赢分（0-2分）
   - < 1.30: 2.0分
   - < 1.50: 1.5分
   - < 1.60: 1.0分
   - >= 1.60: 0分

4. 走势分（0-2分）
   - 赔率下降: +10 * |下降幅度|
   - 最高2分
```

## 🔧 模块详解

### 1. bot.py - 主程序
**职责：**
- 程序初始化和主循环
- 协调各模块工作
- 执行投注和日志记录

**关键函数：**
- `get_balance()`: 获取账户余额
- `place_bet()`: 执行投注
- `init_log_file()`: 初始化日志文件
- `has_bet_on_event()`: 检查是否已投注

### 2. matcher.py - 比赛筛选
**职责：**
- 从API获取比赛数据
- 13层过滤筛选
- 赔率走势监控
- 评分排序

**关键函数：**
- `fetch_events()`: 获取比赛数据
- `filter_matches()`: 主筛选流程
- `get_asian_handicap_odds()`: 提取让球盘赔率
- `get_home_win_odds()`: 提取独赢赔率
- `analyze_odds_trend()`: 走势分析
- `calculate_match_score()`: 计算评分

### 3. bankroll.py - 资金管理
**职责：**
- 2.5倍马丁格尔计算
- 连败检测
- 日内风控

**关键函数：**
- `calc_base_stake()`: 计算首注金额
- `next_bet_amount()`: 计算下一注金额
- `current_loss_streak()`: 计算连败次数
- `calc_today_pnl()`: 计算今日盈亏

### 4. config.py - 配置管理
**职责：**
- 集中管理所有配置参数

**配置分类：**
- API配置: API_KEY, BASE_URL, BET_URL
- 筛选配置: TARGET_HANDICAP, ODDS范围, 时间窗口
- 资金配置: R, Q, N, MAX_ONE, MIN_BET
- 风控配置: DAILY_LOSS_LIMIT, DAILY_PROFIT_TARGET
- 系统配置: DRY_RUN, SLEEP_INTERVAL

### 5. core_leagues.py - 联赛数据库
**职责：**
- 定义3层联赛体系
- 联赛评分映射

**数据结构：**
- TIER_1_LEAGUES: 38个核心联赛（五大联赛+主流）
- TIER_2_LEAGUES: 115个扩展联赛（次级+地区）
- TIER_3_LEAGUES: 31个国际赛事
- LEAGUE_SCORES: 联赛评分映射

## ⚠️ 风险控制

### 1. 投注层面
- 单场最大5%资金
- 连败4次后停止
- 30分钟最多1场

### 2. 日内层面
- 单日亏损20%停止
- 单日盈利5%可选择收工

### 3. 赔率层面
- 让球盘：1.2-1.95
- 独赢：< 1.8
- 走势：尾盘必须下降

### 4. 联赛层面
- 默认只投注TIER_1（38个核心联赛）
- 可扩展到EXPANDED（153个）
- 建议胜率>60%后再扩展

## 📈 优化建议

### 1. 时间窗口
- 初始建议: 5-10分钟
- 经验数据: 2-3分钟窗口赔率更稳定
- 调试工具: debug_matcher.py

### 2. 赔率走势
- 初期建议: 禁用（ENABLE_ODDS_TRACKING=False）
- 积累数据后: 启用并设置阈值
- 优化方向: 尾盘下降>0.03最佳

### 3. 联赛模式
- 测试阶段: TIER_1（38个）
- 稳定运行: EXPANDED（153个）
- 激进模式: ALL（184个，不推荐）

### 4. 评分系统
- 设置MIN_MATCH_SCORE筛选低质量比赛
- 建议值: 8.0-10.0

## 🔐 安全建议

1. **API密钥管理**
   - 不要上传到公共仓库
   - 使用环境变量或独立配置文件

2. **资金安全**
   - 初期使用小额资金测试
   - 启用DRY_RUN模拟运行
   - 设置严格的单日亏损限制

3. **监控运行**
   - 定期查看日志
   - 监控胜率和盈亏
   - 及时调整参数
