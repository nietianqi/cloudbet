# 🔧 Bug修复说明

## 问题描述

**症状：** "有很多符号的比赛 为什么没有成交"
- 系统显示找到比赛，但从不下注
- 修改 `MATCH_TIME_BEFORE` 配置无效

## 根本原因

**matcher.py 第303行存在硬编码值：**

```python
# ❌ 错误代码（已修复）
if not (1.5 <= time_to_match <= 2.5):
    continue
```

**问题：**
- 无论 `config.py` 中 `MATCH_TIME_BEFORE` 设为多少，系统都只查找**离开赛1.5-2.5分钟**的比赛
- 当你改为 `MATCH_TIME_BEFORE = 10` 时，代码仍然使用 1.5-2.5 分钟的硬编码值

## 修复内容

### 1. matcher.py (lines 303-306)

```python
# ✅ 修复后
time_min = MATCH_TIME_BEFORE - 0.5
time_max = MATCH_TIME_BEFORE + 0.5

if not (time_min <= time_to_match <= time_max):
    # 如果开赛前10分钟，记录赔率用于走势分析
    if ENABLE_ODDS_TRACKING and time_to_match <= TRACKING_START_MINUTES:
        ahc_odds = get_asian_handicap_odds(match_info["markets"])
        if ahc_odds:
            track_odds(event_id, ahc_odds)
    continue
```

### 2. debug_matcher.py (lines 110-117)

```python
# ✅ 修复后
time_min = MATCH_TIME_BEFORE - 0.5
time_max = MATCH_TIME_BEFORE + 0.5

if not (time_min <= time_to_match <= time_max):
    filtered_by_time += 1
    print(f"   ⏰ {match_name}")
    print(f"      开赛: {time_to_match:.1f}分钟后 (需要{time_min:.1f}-{time_max:.1f}分钟)")
    continue
```

## 现在的行为

### 当前配置 (config.py)
```python
MATCH_TIME_BEFORE = 10  # 开赛前10分钟
```

### 实际时间窗口
```
time_min = 10 - 0.5 = 9.5 分钟
time_max = 10 + 0.5 = 10.5 分钟
```

**系统会在以下情况下注：**
- 比赛离开赛 **9.5 到 10.5 分钟** 之间
- 符合其他条件（-1盘口、赔率范围、独赢赔率等）

## 其他注意事项

### 1. 时间窗口的含义

```
MATCH_TIME_BEFORE = 10 表示:
  ┌──────────────────┬────┬──────────────────┐
  │                  │窗口│                  │
  比赛开始前          10.5 9.5              现在
                     分钟 分钟

在这个窗口内（9.5-10.5分钟）才会下注
```

### 2. 为什么是 ±0.5 分钟？

- API 每30秒刷新一次
- ±0.5 分钟 = ±30秒 的容错
- 确保不会因为几秒钟的延迟错过比赛

### 3. 轮询频率

当前配置：
```python
SLEEP_INTERVAL = 30  # 每30秒检查一次
```

**计算：**
- 10分钟窗口 = 600秒
- 30秒扫描一次 = 600/30 = 20次扫描
- **足够捕获到窗口内的比赛**
