## ⚙️ 核心策略

### 固定不变的投注目标

1. ✅ 只玩：**主队亚洲让一球（-1）盘口**
2. ✅ 赔率区间：**1.50 - 1.95**
3. ✅ 时间窗口：**离开赛 2 分钟**
4. ✅ 核心联赛：**五大联赛 + 主流联赛**

### 比赛筛选（多维度）

#### 第一层：基础筛选
- ✅ 让球盘 = -1
- ✅ 赔率范围：1.50 - 1.95
- ✅ 更严格的安全区：1.55 - 1.80
- ✅ 开赛时间：2 分钟 ± 30秒

#### 第二层：联赛筛选
- ✅ 只投注 TIER_1 核心联赛（15个）
- ✅ 五大联赛、巴甲、日职联等信息最充分的联赛
- ✅ 自动排除虚拟联赛

#### 第三层：真·强队筛选
- ✅ 主队独赢（1X2）赔率 < 1.60
- ✅ 确保是市场认可的大热门
- ✅ 避免"半斤八两硬抬成-1"的比赛

#### 第四层：赔率走势分析（可选）
- ✅ 从开赛前 10 分钟开始监控赔率
- ✅ 要求尾盘下降至少 0.03（走热）
- ✅ 波动不能超过 0.15（避免剧烈震荡）
- ✅ 过滤掉"赔率上升"的比赛

#### 第五层：场次控制
- ✅ 每 30 分钟最多下注 1 场
- ✅ 多场候选时，选评分最高的
- ✅ 评分低于阈值直接放弃

### 评分系统

每场比赛综合评分（0-20分）：

```python
总分 = 联赛分(0-10) + 赔率分(0-3) + 独赢分(0-2) + 走势分(0-2)
```

**评分因素：**
1. 联赛等级：五大联赛 10分，主流联赛 7-8分
2. 让球盘赔率：越接近 1.70 分数越高
3. 独赢赔率：越低（强队）分数越高
4. 赔率走势：下降越多分数越高

## 💰 资金管理（2.5倍马丁格尔）

### 核心参数

```python
R = 0.30        # 单序列占总资金 30%
Q = 2.5         # 倍率 2.5
N = 4           # 最大连输 4 次
MAX_ONE = 0.05  # 单场最大 5%
```

### 公式

**首注金额：**

```
s0 = R × B × (Q - 1) / (Q^(N+1) - 1)
```

**第 k 次投注：**

```
stake = s0 × Q^k
```

**限制条件：**

```
stake = max(stake, 1.0)          # 最小 1 USDT
stake = min(stake, B × 0.05)     # 最大 5%
```

### 资金需求示例

假设余额 **100 USDT**，配置 `R=0.3, Q=2.5, N=4`：

| 轮次 | 连败 | 投注金额 | 累计投注 | 占比 |
|------|------|---------|---------|------|
| 1    | 0    | 0.47    | 0.47    | 0.5% |
| 2    | 1    | 1.16    | 1.63    | 1.6% |
| 3    | 2    | 2.91    | 4.54    | 4.5% |
| 4    | 3    | 5.00    | 9.54    | 9.5% |
| 5    | 4    | 5.00    | 14.54   | 14.5% |

**最坏情况：**连败 5 次，损失 14.54 USDT（14.5%）

| 配置 | 最小本金 | 推荐本金 | 说明 |

| 当前 | 100 USDT | 200 USDT | 平衡型 |

## 🛡️ 风险控制

### 日内风控（自动触发）

1. **单日最大亏损：20%**
   - 达到后停止当天所有投注

2. **单日盈利目标：5%**
   - 达到后提醒收工（可配置是否自动停止）

3. **连败上限：4 次**
   - 超过后强制结束序列，停止投注

### 余额保护

- 单场投注不超过余额 5%
- 始终保留至少 1 USDT 余额
- 余额不足时自动停止

## 📊 配置参数说明

### config.py 核心参数

```python
# === 比赛筛选 ===
TARGET_HANDICAP = "-1"          # 让球数（固定）
TARGET_ODDS_LOW = 1.50          # 最低赔率
TARGET_ODDS_HIGH = 1.95         # 最高赔率
SAFE_ODDS_LOW = 1.55            # 安全区下限（推荐）
SAFE_ODDS_HIGH = 1.80           # 安全区上限（推荐）
MAX_HOME_WIN_ODDS = 1.60        # 独赢上限

# === 赔率监控 ===
ENABLE_ODDS_TRACKING = True     # 启用走势监控
TRACKING_START_MINUTES = 10     # 提前10分钟开始
ODDS_DROP_THRESHOLD = -0.03     # 要求下降0.03

# === 资金管理 ===
R = 0.30                        # 序列占比 30%
Q = 2.5                         # 倍率 2.5
N = 4                           # 最大连败 4
MAX_ONE = 0.05                  # 单场最大 5%

# === 风控 ===
DAILY_LOSS_LIMIT = 0.20         # 单日亏损 20%
DAILY_PROFIT_TARGET = 0.05      # 单日盈利 5%

# === 系统 ===
SLEEP_INTERVAL = 30             # 轮询间隔 30秒
DRY_RUN = False                 # 模拟模式
```

### bet_log.csv 字段说明

| 字段 | 说明 |
|------|------|
| Timestamp | 投注时间（UTC） |
| EventID | 比赛ID |
| Match | 比赛名称 |
| League | 联赛 |
| AHC_Odds | 让球盘赔率 |
| HomeWin_Odds | 独赢赔率 |

| LossStreak | 连败次数 |
| Result | 结果（ACCEPTED/FAILED/WIN/LOSE） |
| PnL | 盈亏（结算后填入） |
| Balance | 余额 |
| Score | 匹配评分 |
| Notes | 备注 |

### 手动更新结果

比赛结束后，手动更新 `Result` 和 `PnL` 字段：

```csv
Result: WIN 或 LOSE
PnL: 盈利金额（正数）或亏损金额（负数）
```

例如：
```
投注 10 USDT @ 1.70，赢了：
  Result = WIN
  PnL = 7.0 (10 × 0.70)

输了：
  Result = LOSE
  PnL = -10.0
```

## ⚠️ 风险警告

### 马丁格尔策略的本质风险

1. **理论上能回本，但需要无限资金**
   - 实际上资金有限，连败会导致爆仓
   - 2.5倍递增比2倍更激进

2. **胜率要求高**
   - 需要至少 **60%** 的真实胜率才能长期盈利
   - 赔率有抽水（5-10%），实际胜率更难达到

3. **足球比赛不可预测**
   - 即使强队让1球也可能输球
   - 点球、红牌、伤病等突发因素
   - 任何筛选策略都不能保证稳赢

4. **连败概率**
   - 假设 50% 胜率，连败 4 次概率：6.25%
   - 假设 55% 胜率，连败 4 次概率：4.1%
   - 假设 60% 胜率，连败 4 次概率：2.6%

### 期望收益率（仅供参考）

## 🔧 常见问题

### Q: 找不到符合条件的比赛？

**原因：**
- 时间窗口太窄（恰好 2 分钟的比赛较少）
- 赔率和联赛限制太严格
- 当前时段比赛少

**解决：**
1. 放宽时间窗口：`MATCH_TIME_BEFORE = 3` 或 5
2. 调整赔率范围：`TARGET_ODDS_HIGH = 2.0`
3. 禁用赔率监控：`ENABLE_ODDS_TRACKING = False`
4. 扩大联赛范围：在 `core_leagues.py` 中添加 TIER_2

### Q: 投注一直失败？

**检查：**
1. API Key 是否正确
2. 余额是否充足
3. 网络连接是否正常
4. 查看 `bot.log` 的错误信息
5. 赔率是否快速变化

### Q: 如何提高胜率？

**尝试：**
1. 收紧赔率范围（如 1.55-1.75）
2. 提高独赢上限要求（如 < 1.50）
3. 只投注表现最好的 5 个联赛
4. 提高最低评分阈值（如 MIN_MATCH_SCORE = 8）
5. 增加更多筛选条件

### Q: 如何模拟测试？

在 `config.py` 中设置：

```python
DRY_RUN = True
```

程序会执行所有筛选逻辑，但不实际下注。
