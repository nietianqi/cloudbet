# 常见问题解决方案

## 🔍 问题1：为什么日志显示 443 端口？

### 解答

**443 端口是正常的，不是错误！**

```
DEBUG] Starting new HTTPS connection (1): sports-api.cloudbet.com:443
```

**说明：**
- 443 是 HTTPS 的标准端口（就像 HTTP 使用 80 端口）
- 所有安全的 API 连接都使用 443
- 这是正常的日志信息，表示建立加密连接

**HTTP vs HTTPS 端口：**
```
HTTP:  http://example.com:80   (不安全)
HTTPS: https://example.com:443 (安全，加密)
```

**结论：** 可以忽略，这是正常的连接日志。

## 🚫 问题2：为什么总是"暂无符合条件的比赛"？

### 根本原因

**时间窗口太窄！**

当前配置：
```python
MATCH_TIME_BEFORE = 2  # 离开赛恰好 2 分钟 (±30秒)
```

实际查询范围：开赛前 **1.5 - 2.5 分钟**

**问题：**
- 符合条件的比赛太少（每小时可能只有 0-2 场）
- 需要恰好在这个时间窗口刷新
- 错过窗口就要等下一批比赛

### 解决方案

#### 方案A：放宽时间窗口（推荐）

```python
# config.py
MATCH_TIME_BEFORE = 5  # 改为 5 分钟
```

**效果：**
- 时间范围：开赛前 4.5 - 5.5 分钟
- 更容易找到比赛
- 仍然是临场下注

#### 方案B：更激进（找到更多比赛）

```python
MATCH_TIME_BEFORE = 10  # 改为 10 分钟
```

**效果：**
- 时间范围：开赛前 9.5 - 10.5 分钟
- 更多投注机会
- 但赔率可能波动更大

#### 方案C：禁用精确时间限制

修改 `matcher.py` 中的时间筛选：

```python
# 原来：恰好 2 分钟 ± 30秒
if not (1.5 <= time_to_match <= 2.5):
    continue

# 改为：任何 2-10 分钟内的比赛
if not (2 <= time_to_match <= 10):
    continue
```

## 📊 问题4：筛选条件太严格怎么办？

### 当前筛选条件

```python
# 时间
MATCH_TIME_BEFORE = 2          # 离开赛 2 分钟

# 赔率
TARGET_ODDS_LOW = 1.5          # 最低 1.5
TARGET_ODDS_HIGH = 1.95        # 最高 1.95
SAFE_ODDS_LOW = 1.55           # 安全区下限
SAFE_ODDS_HIGH = 1.8           # 安全区上限

# 独赢
MAX_HOME_WIN_ODDS = 1.6        # 独赢赔率 < 1.6

# 评分
MIN_MATCH_SCORE = 5.0          # 最低评分 5 分
```

### 逐步放宽策略

#### 第一步：放宽时间（优先）

```python
MATCH_TIME_BEFORE = 5  # 或 10
```

#### 第二步：放宽赔率安全区

```python
SAFE_ODDS_LOW = 1.50           # 等于最低赔率
SAFE_ODDS_HIGH = 1.95          # 等于最高赔率
```

#### 第三步：放宽独赢限制

```python
MAX_HOME_WIN_ODDS = 1.70       # 或 1.80
```

#### 第四步：禁用赔率走势监控（临时）

```python
ENABLE_ODDS_TRACKING = False
```

#### 第五步：降低评分要求

```python
MIN_MATCH_SCORE = 3.0          # 或 0（不限制）
```

### 推荐配置（找到更多比赛）

```python
# config.py - 放宽版

# 时间：改为 5-10 分钟
MATCH_TIME_BEFORE = 5

# 赔率：使用全范围
SAFE_ODDS_LOW = 1.50
SAFE_ODDS_HIGH = 1.95

# 独赢：放宽
MAX_HOME_WIN_ODDS = 1.70

# 禁用走势监控（先积累数据）
ENABLE_ODDS_TRACKING = False

# 降低评分
MIN_MATCH_SCORE = 0
```

## 📝 总结

| 问题 | 原因 | 解决方案 |
|------|------|---------|
| **443端口** | HTTPS标准端口 | 正常，忽略 |
| **找不到比赛** | 时间窗口太窄 | 改为5-10分钟 |
| **条件太严** | 多层筛选 | 逐步放宽 |
| **错过比赛** | 扫描频率低 | 高峰时段运行 |

1. 编辑 `config.py`：
   ```python
   MATCH_TIME_BEFORE = 10
   LEAGUE_MODE = 'EXPANDED'
   ENABLE_ODDS_TRACKING = False
   ```

2. 重启程序：
   ```bash
   python bot.py
   ```

3. 观察效果，根据需要调整

### 调试步骤

1. 运行调试工具：`python debug_matcher.py`
2. 查看筛选统计
3. 根据建议调整配置
4. 重新测试
