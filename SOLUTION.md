## ❌ 问题根源

### 1. 主要瓶颈：-1盘口稀少（最严重）

**问题：**
- 69场比赛中，16场（23%）没有 handicap=-1 的盘口
- Cloudbet 不是所有比赛都提供-1盘口

**为什么-1盘口少？**
- 只有实力差距明显的比赛才会开-1盘口
- 平局较多的联赛（如日职联）倾向于开-0.5或-0.75
- 比赛临近时，一些盘口会关闭

**影响：**
- 即使其他条件都满足，没有-1盘口就无法下注
- 这是当前配置的最大瓶颈

---

### 2. 时间窗口过窄

**当前配置：**
```python
MATCH_TIME_BEFORE = 1  # 只查找0.5-1.5分钟窗口
```

**问题：**
- 窗口太小（1分钟），很难精确捕获
- 需要比赛恰好在0.5-1.5分钟后开赛
- 8场比赛因时间窗口被过滤

**为什么设计这么窄？**
- 原设计目的：尾盘赔率更稳定
- 但实际：捕获率太低

---

### 3. 当前时段问题

**现在时间：** 北京时间 23:09

**问题：**
- 欧洲主流联赛通常在北京时间 23:00-04:00 开赛
- 当前时段比赛较少，且多为低级别联赛
- 低级别联赛-1盘口更少

## ✅ 解决方案（优先级排序）

### 方案1：支持更多盘口类型（推荐，长期）

**修改：** 不仅限于-1，支持 -0.5, -0.75, -1.25 等

**优点：**
- 大幅增加可投注比赛数量
- -0.75盘口也很常见且稳定
- 更灵活

**实施：**
```python
# config.py
TARGET_HANDICAPS = ["-1", "-0.75", "-1.25"]  # 支持多个盘口

# matcher.py 修改筛选逻辑
for handicap in TARGET_HANDICAPS:
    odds = get_asian_handicap_odds(markets, handicap)
    if odds:
        break
```

---

**修改：**

- 赔率仍然相对稳定

**缺点：**
- 赔率可能稍有波动（可接受）

**最佳时段：**

**缺点：**

## 🔧 快速修改脚本

保存为 `fix_config.py` 并运行：

```python
import re

# 读取配置
with open('config.py', 'r', encoding='utf-8') as f:
    content = f.read()

# 修改时间窗口
content = re.sub(
    r'MATCH_TIME_BEFORE\s*=\s*\d+',
    'MATCH_TIME_BEFORE = 5',
    content
)

# 修改盘口（可选）
content = re.sub(
    r'TARGET_HANDICAP\s*=\s*"-1"',
    'TARGET_HANDICAP = "-0.5"',
    content
)

# 修改赔率范围（配合-0.5）
content = re.sub(
    r'TARGET_ODDS_LOW\s*=\s*[\d.]+',
    'TARGET_ODDS_LOW = 1.5',
    content
)

# 写回
with open('config.py', 'w', encoding='utf-8') as f:
    f.write(content)

print("✅ 配置已修改！")
print("  - 时间窗口: 1 → 5分钟")
print("  - 盘口: -1 → -0.5")
print("  - 赔率范围: 1.2 → 1.5")
```
